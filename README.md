# 拼豆摆豆助手（Pindou-help）

一个**纯前端微信小程序**：上传现成的拼豆图纸，自动识别豆数与色号，用「单色高亮 + 网格坐标」的方式辅助你快速摆豆。

灵感来自 [fuse-bead-tool](https://github.com/atonasting/fuse-bead-tool)，但思路相反——

> 原工具解决的是「**怎么把图片变成图纸**」；
> 本工具解决的是「**图纸已经有了，怎么摆得更快**」。

---

## 功能一览

- 📤 **上传拼豆图纸**：支持送拼豆 App、小红书等渠道下载的标准图纸（带网格线 / 坐标轴 / 格内色号文字 / 底部图例）
- 🔍 **自动网格识别**：识别图纸是几乘几的豆子矩阵，红网格叠图预览，支持手动微调
- 🎨 **Mard 官方色号映射**：识别结果映射到 24 / 48 / 72 / 96 / 120 / 144 / 168 / 221 八档官方豆盘，色卡直接显示真实色号（H7、E11…）和豆数
- 🔦 **单色高亮**：点色卡或点图纸格子，只高亮该色号的豆子，其余变暗——拿着一种豆，一口气摆完
- 🧭 **网格叠加层**：细网格线 + 每 5 格粗线 + 屏幕边缘行列坐标 + 放大后格内色号文字
- 👆 **手势操作**：单指拖动、双指捏合缩放（锚点跟随）
- 🌙 **深 / 浅主题**：默认深色（摆豆环境光线杂，深色更护眼），选择持久化

## 设计思想

### 1. 做减法，把一件事做透

拼豆链路是「选图 → 生成图纸 → 买豆 → 摆豆 → 熨烫」。「生成图纸」已经有送拼豆 App、Pixelbead 等成熟工具，用户真正痛的是**摆豆环节**：对着手机小屏幕数格子，摆错、串行、数不清。

所以本工具砍掉图片转图纸的全部功能（颜色量化、抖动、色板拟合），只保留并强化原工具里的一个辅助功能——**单色显示**，把它做成主角。

### 2. 图纸不是图片，是「带标注的矩阵」

一张标准拼豆图纸 ≠ 像素图。它是：豆子矩阵（每颗豆约 9×9 px）+ 网格线 + 行列坐标 + 格内色号文字 + 底部图例。直接按像素处理必死，所以识别分四步：

1. **网格检测**：对灰度图做列/行边缘强度投影，网格线横贯图幅会形成周期性尖峰 → 峰间距得 pitch、首峰得 offset、峰数得豆数
2. **校准兜底**：自动识别结果用红网格叠图给用户确认，宽/高可步进微调——算法负责 90% 的场景，人负责兜底
3. **中心采样**：每颗豆取格子中心一小块区域的**众数色**（5bit 量化统计），避开格内文字和网格线的干扰
4. **空白判定**：近白格子扫描格内深色墨迹——有色号文字的是白豆（如 H2），没文字的才是真空白

### 3. 色号映射到真实豆盘，而不是抽象颜色

用户手里拿的是 Mard 实体豆，图纸上写的也是 Mard 色号。识别出的颜色必须落到**用户真实拥有的豆盘**上：

- 内置 **24 ~ 221 八档官方套装**色号数据（严格嵌套，24 ⊂ 48 ⊂ … ⊂ 221）
- 采样色按 **CIE Lab 感知色差**匹配豆盘内最近色号（比 RGB 距离更接近人眼）
- 用户选自己的豆盘档位，工具只显示他买得到的颜色

### 4. 渲染要扛得住大图，还要读得懂

- **混合渲染**：总览时整图走离屏位图 `drawImage`（O(1)，25 万格不卡）；放大后逐格绘制，豆子画成圆形更像实物
- **网格叠加层**：摆豆是空间工作，必须给坐标系——细线分格、5 格粗线分段、行列数字钉在屏幕边缘随滚动跟随、放到最大格内直接显示色号
- **流式布局**：小程序 canvas 是原生组件，会盖住 WXML 的普通 view，所以顶栏 / 画布 / 面板纵向流式排列，不用 `fixed` 叠层

### 5. 性能与体验细节

- 调色板 / 网格索引用 `Int32Array`，不进 `setData`（避免大数组序列化）
- 只渲染可见区域的格子；色号匹配用 6bit 量化缓存
- 上传图自动限制边长（≤800px），读取像素后原图即弃
- 主题、豆盘档位均持久化

## 技术栈

原生微信小程序（WXML / WXSS / JS + Canvas 2D），零依赖、无后端、无需服务器域名。

## 项目结构

```
├── project.config.json
└── miniprogram/
    ├── app.js / app.json / app.wxss
    ├── pages/index/          # 唯一页面：上传 → 校准 → 摆豆 三阶段
    │   ├── index.js          # 网格识别、采样、色号映射、渲染、手势
    │   ├── index.wxml
    │   └── index.wxss
    └── utils/
        └── palette.js        # Mard 八档套装色号 + Lab 色差匹配
```

## 本地运行

1. 安装[微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 导入本仓库根目录，AppID 选「测试号」
3. 编译运行，上传一张拼豆图纸即可体验
